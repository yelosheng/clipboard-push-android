## 3. The "Pitfall" Registry (Troubleshooting)

### Environment & Dependencies
- **Proxy Issues**: `vcpkg` download failures (exit code 87) were resolved by clearing inconsistent `HTTP_PROXY`/`HTTPS_PROXY` environment variables during the build session.
- **Broken Port**: The `socket-io-client-cpp` port in `vcpkg` has broken header paths on Windows.
  - *Solution*: Stubbed out the library and rewrote the logic using `Qt6::WebSockets`.

### Windows API & Headers
- **Missing Identifiers**: Functions like `DragQueryFile` and `SetCurrentProcessExplicitAppUserModelID` required specific headers not included in standard `windows.h`.
  - *Fix*: Added `#include <shellapi.h>` and `#include <shobjidl.h>`.

### Runtime & Deployment
- **Platform Plugin Error**: "No Qt platform plugin could be initialized."
  - *Fix*: Created `platforms/` directory and copied `qwindows.dll`.
- **SSL/TLS Support**: "SSL Sockets are not supported on this platform."
  - *Fix*: Copied OpenSSL DLLs (`libcrypto`, `libssl`) and the TLS backend plugin `tls/qopensslbackend.dll`.
- **Image Clipboard Failure**: Images received but not copied to clipboard in Release mode.
  - *Fix*: Added `imageformats/` plugin directory to support JPEG/GIF formats.
